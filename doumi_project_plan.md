# 도움이 (Doumi) — 프로젝트 기획서

> 디지털 소외계층을 위한 AI 웹 대리인 서비스  
> SDG 10 · 불평등 감소 | 교육/컴퓨터 분야 연계 웹 프로젝트

---

## 1. 프로젝트 개요

### 서비스명
**도움이 (Doumi)**

### 핵심 컨셉
고령층·저소득층이 공공 웹서비스(정부24, 건강보험공단, 코레일 등)를 이용하다 막히는 순간을 AI가 실시간으로 감지하고, 사용자 동의 후 직접 대신 처리해주는 서비스.

단순 정보 안내나 UI 개선이 아니라, **사용자가 실제로 완료하지 못하는 행정·생활 서비스를 AI 에이전트가 직접 실행해 완결**시키는 것이 핵심.

### SDG 연계
- **SDG 10 (불평등 감소)**: 디지털 격차가 복지·의료·이동권 격차로 이어지는 구조적 불평등 해소
- **교육/컴퓨터 분야**: Chrome 익스텐션 + 웹 에이전트 + AI API 통합 기술 활용

### 대상 사용자
- 고령층 (60대 이상): 공공 웹서비스 이용 시 반복적 실패 경험
- 저소득층: 디지털 소외로 복지 사각지대 발생
- 보조 사용자: 고령자 가족·사회복지사 (설치 및 초기 세팅 도움)

## 2. 핵심 문제 정의

### 실제 막히는 순간들

| 서비스 | 막히는 지점 | 실제 피해 |
|--------|------------|---------|
| 정부24 기초연금 신청 | 검색 결과 17개 중 어떤 걸 눌러야 할지 모름 + 간편인증 화면에서 멈춤 | 연 최대 수십만원 복지 미수령 |
| 건강보험공단 | '민원여기요' 메뉴 구조 이해 불가, 보험료 감면 대상임에도 신청 경로 모름 | 연 최대 36만원 감면 미신청 |
| 코레일 KTX 예매 | 달력 UI 실패 반복, 결제 단계 이탈 | 현장 창구 또는 가족 도움 의존 |

### 핵심 인사이트
막힘의 유형이 세 가지로 압축됨:
1. **메뉴 구조를 못 읽는 것** — 용어·분류 체계 낯섦
2. **용어를 모르는 것** — 간편인증, 세대 보험료, 차상위 등
3. **인증·결제 단계에서 이탈** — 공인인증서, 카드번호 입력 등

## 3. 서비스 구조

### 플랫폼 구성
웹사이트 단독으로는 Same-Origin Policy(SOP)로 인해 타 사이트 DOM 조작이 원천 불가. 따라서 **웹사이트 + Chrome 익스텐션 조합**으로 구현.

```
[사용자 브라우저]
    Chrome 익스텐션 (행동 감지 + DOM 조작 + 음성)
        ↕ localhost:7878
    OpenClaw (로컬 설치, Skills 실행 런타임)

        ↕ HTTPS API 호출 (API 키 보관)
[도움이 웹 백엔드 - Next.js]
    /api/analyze   → Claude API (막힘 판단)
    /api/vision    → GPT-4o Vision (화면 분석)
    /api/whisper   → Whisper API (음성 인식)
    /api/tts       → OpenAI TTS (음성 출력)

[공공 사이트]
    정부24 / 건강보험공단 / 코레일 등
```

### 레포지토리 구성 (3개)

#### 1. doumi-extension
Chrome 익스텐션. 사용자 브라우저 위에서 동작.
- 행동 감지 엔진 (detector.ts)
- 개입 UI 오버레이 (overlay.ts)
- 음성 입력·출력 (voice.ts)
- OpenClaw 통신 브릿지 (openclaw-bridge.ts)

#### 2. doumi-web
Next.js 웹사이트. **회원가입·로그인 없음.**
- 서비스 소개 + 설치 안내 원페이지
- OS 감지 + 설치 파일(.exe/.dmg) 다운로드
- API Route (Claude·OpenAI API 키 중계)
- 디바이스 토큰 발급 및 rate limiting
- Supabase 연동 (토큰 관리만)

#### 3. doumi-skills
OpenClaw Skills 레포.
- 서비스별 자동화 스크립트 (gov24, nhis, korail)
- Vision fallback 모듈
- 공유 유틸리티

## 4. 핵심 기능 명세

### 기능 1 — 실시간 행동 이상 감지 엔진

**개요**
사용자가 공공 웹사이트에서 막히는 순간을 실시간으로 포착하는 JavaScript 이벤트 엔진.

**감지 패턴**
- 마우스 3초 이상 정체 (idle detection)
- 동일 버튼·메뉴 2회 이상 반복 클릭
- 스크롤 없이 5초 이상 화면 고정
- 뒤로 가기 후 동일 페이지 재진입

**기술 구현**
```typescript
// detector.ts 핵심 로직
const IDLE_THRESHOLD = 3000;      // 3초 정체
const REPEAT_CLICK_THRESHOLD = 2; // 동일 요소 2회 클릭
const SCROLL_IDLE_THRESHOLD = 5000; // 스크롤 없이 5초

// 행동 시퀀스를 Claude API로 전송 → 막힘 여부 판단
```

**차별점**
단순 타이머가 아닌 행동 시퀀스 조합. "3초 정체 + 반복 클릭 + 스크롤 없음"이 겹칠 때만 개입. 그냥 읽고 있는 상태와 막힌 상태를 구분.

---

### 기능 2 — AI 개입 판단 및 동의 인터페이스

**개요**
행동 이상 감지 시 Claude API가 현재 페이지 컨텍스트를 분석해 개입 여부를 판단하고, 사용자 동의를 받는 UX.

**Claude API 입력 구조**
```json
{
  "page_url": "https://www.gov.kr/...",
  "page_title": "기초연금 신청",
  "dom_snapshot": "<현재 페이지 DOM 요약>",
  "behavior_sequence": [
    { "type": "idle", "duration": 4200, "element": ".search-result-list" },
    { "type": "click", "count": 2, "element": ".menu-item-01" }
  ]
}
```

**개입 UI 원칙**
- 화면 하단 고정 바 형태 (팝업·모달 아님)
- 텍스트: "기초연금 신청을 도와드릴까요?"
- 버튼: [네, 도와주세요] [아니오] — 두 개만, 최소 60px 높이
- 음성으로도 동시 안내

---

### 기능 3 — OpenClaw 웹 에이전트 자동 처리

**개요**
사용자 동의 후 OpenClaw가 브라우저를 직접 제어해 서비스 신청·조회·예매를 자동 처리.

**처리 방식**
1. 익스텐션 background → `localhost:7878` → OpenClaw에 Skill 실행 명령
2. OpenClaw가 해당 서비스 Skill 스크립트 실행
3. GPT-4o Vision으로 현재 화면 분석 → 버튼·입력창 위치 파악
4. DOM 조작으로 폼 자동 입력
5. 최종 제출 버튼은 **반드시 사용자가 직접 클릭** (법적 책임 원칙)

**지원 서비스 (MVP)**

| 서비스 | Skill 파일 | 주요 처리 내용 |
|--------|-----------|-------------|
| 정부24 기초연금 | gov24/pension-apply | 신청 메뉴 탐색 → 간편인증 안내 → 신청서 작성 |
| 건강보험공단 | nhis/insurance-check | 보험료 조회 → 감면 대상 확인 → 신청 연결 |
| 코레일 KTX | korail/ktx-booking | 출발·도착·날짜 입력 → 시니어 할인 적용 → 좌석 선택 |

**Vision Fallback**
셀렉터가 사이트 변경으로 깨졌을 때, GPT-4o Vision이 스크린샷을 보고 버튼 위치를 실시간 판단. 하드코딩 의존도 최소화.

---

### 기능 4 — 음성 기반 입력 인터페이스 (STT)

**개요**
타이핑이 어려운 고령층을 위해 음성으로 원하는 작업을 직접 요청할 수 있게 설계.
TTS(음성 출력) 기능은 Phase 3 고도화 단계에서 추가 예정.

**음성 입력 파이프라인**
```
사용자 발화
→ Web Audio API (브라우저 마이크 녹음)
→ 익스텐션 background
→ doumi-web /api/whisper
→ OpenAI Whisper API (한국어 변환)
→ Claude API (의도 파악)
→ OpenClaw Skill 실행
```

**음성 인식 확인 UX**
- 인식 결과를 화면에 크게 표시
- "이렇게 말씀하셨나요?" 확인 단계
- 틀리면 다시 말하기 가능

**MVP 음성 안내 방식**
TTS 고도화 전까지는 화면 텍스트로 단계 안내.
- 개입 제안: 하단 바 텍스트로 표시
- 처리 중: "2/5단계 진행 중" 진행 표시바
- 완료: 완료 화면 + 접수번호 크게 표시

---

### 기능 5 — 능동 요청 모드 (사용자 직접 호출)

**개요**
AI가 막힘을 감지해서 먼저 개입하는 것(수동 모드)과 달리, 사용자가 원하는 순간에 직접 도움을 요청하고 작업을 지시하는 모드.
감지를 기다릴 필요 없이 어떤 화면에서든 즉시 AI 에이전트를 호출할 수 있음.

**진입 방법**
- 화면 우측 하단에 항상 노출되는 **"도움 요청" 플로팅 버튼** (최소 64px, 고대비)
- 클릭 시 입력 패널 등장 — 텍스트 입력 또는 마이크 버튼

**사용 시나리오**
```
사용자: [도움 요청 버튼 클릭]
      → "기초연금 신청하고 싶어요" 입력 (텍스트 or 음성)
      → Claude가 현재 페이지 분석 + 의도 파악
      → "지금 정부24에 계시네요. 기초연금 신청을 바로 도와드릴까요?"
      → [네] 클릭 → OpenClaw Skill 실행
```

**지시 가능한 작업 유형**
- 특정 메뉴·페이지 찾기: "보험료 조회 어디서 해요?"
- 폼 자동 입력 요청: "이 신청서 채워줘"
- 현재 화면 설명 요청: "이 화면이 뭔지 설명해줘"
- 다음 단계 안내: "이제 뭘 눌러야 해?"
- 특정 서비스 직접 실행: "KTX 예매해줘"

**기술 구현**
```typescript
// overlay.ts — 플로팅 버튼 + 입력 패널
// 어떤 사이트에서든 content script가 주입되면 버튼 노출

// 사용자 입력 → Claude API로 전송
{
  "mode": "active_request",         // 능동 요청 모드
  "user_input": "기초연금 신청하고 싶어요",
  "page_url": "https://www.gov.kr/...",
  "dom_snapshot": "<현재 페이지 DOM>",
  "screenshot": "<base64 이미지>"   // GPT-4o Vision 분석용
}
```

**수동 모드와의 차이**

| 구분 | 수동 모드 (감지) | 능동 모드 (직접 호출) |
|------|---------------|-------------------|
| 트리거 | AI가 막힘 감지 후 개입 제안 | 사용자가 버튼 클릭으로 직접 시작 |
| 시점 | 막힌 이후 | 원하는 순간 언제든 |
| 입력 | 없음 (AI가 상황 파악) | 텍스트 또는 음성으로 작업 지시 |
| 적합한 상황 | 어디서 막혔는지 모를 때 | 뭘 하고 싶은지 알고 있을 때 |

### 기능 6 — 고령자 최적화 UX

**디자인 원칙**
- 본문 최소 20px, 버튼 최소 60px 높이
- 흰 배경 + 진한 텍스트 고대비 (#1a1a1a on #ffffff)
- 한 화면에 하나의 질문·하나의 행동만
- 뒤로 가기 + "처음부터 다시" 항상 노출
- 전문 용어·회색 소글씨·팝업 없음

**완료 피드백**
- 큰 글씨 + 음성으로 명확한 완료 신호
- 완료 화면 캡처 저장 기능 (증거 자료)
- 접수번호·예약번호 별도 표시

## 5. 기술 스택

| 영역 | 기술 | 선택 이유 |
|------|------|---------|
| 프론트엔드 | Next.js 14 + TypeScript | App Router, Vercel 배포, 고령자 UX 커스텀 |
| 익스텐션 | Chrome Extension Manifest V3 | 공식 SOP 우회, DOM 접근 합법적 허가 |
| 행동 감지 | 커스텀 JS 이벤트 모듈 | 핵심 로직 직접 구현, 외부 의존 최소화 |
| AI 판단 | Claude API (claude-sonnet-4-6) | 한국어 공공서비스 이해도, Tool use 구조화 |
| 화면 분석 | GPT-4o Vision | 스크린샷 기반 UI 요소 파악, 사이트 변경 대응 |
| 음성 입력 | OpenAI Whisper API | 한국어 노인 발화·사투리 인식률 우수 |
| 음성 출력 | OpenAI TTS | 자연스러운 한국어, 속도·음량 조절 |
| 웹 에이전트 | OpenClaw + Skills | 브라우저 제어 런타임, Skill 기반 확장 구조 |
| DB / 인증 | Supabase | RLS 보안, 무료 티어 MVP 충분 |
| 배포 | Vercel | Next.js 최적 환경, Edge Function |

### API 키 보안 원칙
- 익스텐션 코드에 API 키 **절대 금지** (누구나 열람 가능)
- 모든 AI 호출: 익스텐션 → 도움이 백엔드 → Claude/OpenAI 순서로 중계
- 익스텐션은 백엔드 URL + 사용자 세션 토큰만 보유

## 6. 설치 온보딩 플로우

### 사용자 설치 경험

```
1. doumi.app 접속 (회원가입 없음)
2. 서비스 소개 확인
3. OS 자동 감지
   ├── Windows → .exe 다운로드 버튼
   ├── macOS   → .dmg 다운로드 버튼
   └── 하단 접기: 터미널 사용자용 curl 한 줄
4. 설치 파일 실행 → 다음·다음·완료
5. Chrome 익스텐션 설치 링크로 자동 이동
6. 익스텐션 설치 완료 → 바로 사용 시작
```

### 설치 파일 내부 동작

```bash
# 설치 파일(.exe/.dmg)이 자동으로 처리하는 것
1. Node.js 설치 여부 확인 → 없으면 자동 설치
2. npm i -g openclaw
3. doumi-skills 레포 클론 (공공서비스 스크립트)
4. OpenClaw에 Skills 등록
5. 로컬 서버 포트 7878 설정 및 실행
6. 고유 디바이스 토큰 발급 → 익스텐션에 저장
   (회원가입 없이 API 키 남용 방지 처리)
7. 시스템 시작 시 자동 실행 등록
   ├── Windows: 레지스트리 등록
   └── macOS: launchd plist 등록
```

### API 키 보안 (회원가입 없는 구조)
회원가입 대신 설치 시 발급되는 **디바이스 토큰**으로 인증.
- 익스텐션이 백엔드 호출 시 디바이스 토큰 헤더 포함
- 백엔드에서 토큰 유효성 확인 + 요청 횟수 제한(rate limiting)
- 토큰 탈취 시 해당 토큰만 차단 가능

### 고령자 시나리오
자녀·복지사가 대신 설치 → 이후 고령자는 익스텐션 플로팅 버튼만으로 사용. 설치는 1회, 이후 별도 계정 관리 없이 바로 사용.

## 9. 기술 도전 과제

### 1. 막힘 vs 생각중 구분
단순 idle 감지와 달리 행동 시퀀스 조합으로 판단. 클릭 위치 반복 + 스크롤 없음 + 체류 시간 3가지가 겹칠 때만 개입 트리거.

### 2. 공공기관 사이트 변경 대응
정부24는 UI 업데이트 빈도 높음. 하드코딩 셀렉터 의존 최소화 → GPT-4o Vision이 스크린샷 보고 실시간 판단하는 구조로 내구성 확보.

### 3. 개인정보 처리 신뢰 확보
주민번호·카드번호 등 민감 정보는 메모리 내 처리만, DB 저장 금지. 에이전트 실행 전 "어떤 정보를 입력할지" 화면에 명시 후 동의. 최종 제출은 반드시 사용자 직접 클릭.

### 4. 노인 음성 인식 정확도
느린 발화·사투리·배경 소음 환경에서 Whisper 정확도 확인 필요. 인식 결과 화면 표시 + "맞나요?" 확인 단계 필수.

### 5. 이 프로젝트만의 기술 차별점
행동 감지 + LLM 판단 + 웹 에이전트(OpenClaw) + 음성 인터페이스를 하나의 파이프라인으로 연결한 구조. 각 기술은 기존에 존재하지만, 디지털 소외계층의 실패 순간을 없앤다는 목적으로 통합한 것이 핵심 기여.
